# tests/unit/ — Unit Tests

## Purpose
Unit tests organized to mirror the `base/langflow/` source structure. Each subdirectory tests the corresponding source module.

## Folder Structure

| Folder | Tests For |
|--------|-----------|
| `api/` | API route handlers (`api/v1/`, `api/v2/`). |
| `base/` | Base class functionality (`base/`). |
| `components/` | Individual component tests. |
| `custom/` | Custom component infrastructure. |
| `events/` | Event manager tests. |
| `exceptions/` | Exception handling tests. |
| `graph/` | Graph engine tests — edges, vertices, state, scheduling. |
| `helpers/` | Helper function tests. |
| `initial_setup/` | Initial setup and starter project tests. |
| `inputs/` | Input type tests. |
| `io/` | I/O schema tests. |
| `schema/` | Schema validation tests. |
| `serialization/` | Serialization tests. |
| `services/` | Service layer tests. |
| `template/` | Template engine tests. |
| `utils/` | Utility function tests. |
