# docs/docs/ — Documentation Content

## Purpose
All Markdown documentation content for the Langflow documentation site, organized into topic categories.

## Folder Structure

| Folder | Description |
|--------|-------------|
| `API-Reference/` | REST API reference and usage examples. |
| `Agents/` | Agent concepts and the Tool Calling Agent component. |
| `Components/` | Component reference documentation — one page per component category. |
| `Concepts/` | Core concepts — flows, components, objects, API, playground. |
| `Configuration/` | Configuration guides — API keys, authentication, CLI, database, global variables, security. |
| `Contributing/` | Contributor guides — how to contribute, component development, testing, telemetry. |
| `Deployment/` | Deployment guides — Docker, GCP, Kubernetes, Railway, Render, Hugging Face Spaces. |
| `Get-Started/` | Getting started — installation, quickstart, welcome page. |
| `Integrations/` | Third-party integration guides — Apify, Arize, Composio, Google, Notion, Nvidia, AssemblyAI, Langfuse, etc. |
| `Starter-Projects/` | Starter project walkthroughs — basic prompting, simple agent, RAG. |
| `Support/` | Support resources. |
| `Tutorials/` | Step-by-step tutorials — blog writer, document QA, math agent, memory chatbot, travel planner. |

## For LLM Coding Agents
- Each `.md` file uses Docusaurus front matter with `title`, `description`, and `sidebar_position`.
- Files are named with a category prefix (e.g., `components-models.md`, `configuration-cli.md`).
