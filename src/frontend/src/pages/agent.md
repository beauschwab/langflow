# pages/ — Top-Level Page Components

## Purpose
Top-level page components mapped to URL routes via React Router. Each page represents a distinct screen in the Langflow application.

## Key Subdirectories

| Folder | Description |
|--------|-------------|
| `FlowPage/` | **Flow editor page** — the React Flow canvas where users build flows. The core editing experience. |
| `MainPage/` | **Main dashboard page** — flow list, folder navigation, search, and grid/list views. The landing page after login. |
| `Playground/` | Standalone playground page for testing flows. |
| `LoginPage/` | User login page. |
| `SignUpPage/` | User registration page. |
| `SettingsPage/` | Settings hub — API keys, global variables, shortcuts, store configuration, messages. |
| `AdminPage/` | Admin panel for user management. |
| `AgentsPage/` | Agent management page. |
| `StorePage/` | Langflow Store browser — community components and flows. |
| `ProfileSettingsPage/` | User profile settings. |
| `DeleteAccountPage/` | Account deletion page. |
| `LoadingPage/` | Loading/splash screen shown during app initialization. |
| `ViewPage/` | Read-only flow view page. |
| `AppInitPage/` | Application initialization wrapper. |
| `AppAuthenticatedPage/` | Authenticated app shell wrapper. |
| `AppWrapperPage/` | Root app wrapper with error boundaries. |
| `DashboardWrapperPage/` | Dashboard layout wrapper. |
